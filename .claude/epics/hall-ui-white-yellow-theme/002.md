# Task: Validate MusicPlayer Component

## Description

Review and update the MusicPlayer component to ensure complete yellow theme integration. Verify that all interactive elements (play button, progress bar, volume controls, shuffle/repeat states) properly inherit the yellow theme from CSS custom properties, and fix any hardcoded blue colors or inline styles that bypass the design token system.

**Component:** `/home/user/sdjd2/components/MusicPlayer.tsx`

This component was recently refactored for mobile-first responsive design (commit eef96dd), so it should already be using CSS custom properties. This task validates theme consistency and addresses edge cases.

## Acceptance Criteria

- [ ] All button elements (play/pause, skip, shuffle, repeat) show yellow on active/hover states
- [ ] Progress bar uses yellow fill color (not hardcoded blue)
- [ ] Volume slider uses yellow for active state
- [ ] No hardcoded blue colors in className or style attributes
- [ ] No blue hex values in inline styles (`style={{ color: '#2563eb' }}`)
- [ ] No Tailwind blue classes (`text-blue-600`, `bg-blue-500`, `hover:bg-blue-700`)
- [ ] SVG icons inherit correct yellow color for active states
- [ ] Hover states transition smoothly to yellow (no jarring color changes)
- [ ] Focus states show yellow outline for keyboard navigation
- [ ] Active/playing state indicators use yellow consistently

## Technical Details

**File to Review:** `/home/user/sdjd2/components/MusicPlayer.tsx`

**Key Areas to Inspect:**

1. **Play/Pause Button:**
   - Verify uses `var(--primary)` or CSS class that inherits yellow
   - Check hover state color
   - Validate active/playing visual feedback

2. **Progress Bar:**
   - Confirm `.progress-fill` or equivalent uses yellow
   - Check scrubbing interaction feedback color
   - Verify time markers don't have hardcoded blue

3. **Control Buttons (Skip, Previous, Shuffle, Repeat):**
   - Validate button active states use yellow
   - Check toggle states (shuffle on/off, repeat modes)
   - Ensure SVG icon fills/strokes inherit color

4. **Volume Control:**
   - Verify volume slider uses yellow for active portion
   - Check mute button active state
   - Validate hover feedback

5. **Common Issues to Fix:**
   ```tsx
   // BAD - Hardcoded blue
   <button className="text-blue-600 hover:bg-blue-100">

   // GOOD - Uses CSS custom property
   <button className="text-primary hover:bg-primary-light">

   // BAD - Inline blue style
   <div style={{ backgroundColor: '#2563eb' }}>

   // GOOD - CSS variable
   <div style={{ backgroundColor: 'var(--primary)' }}>
   ```
