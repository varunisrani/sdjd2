# Task 5: Design System Foundation

## Description

Update globals.css with spacing tokens, typography scale, responsive utilities, and animation helpers. This foundational work establishes the design system that all other components will depend on.

## Acceptance Criteria

- [x] 8px spacing system implemented (--spacing-1 through --spacing-8)
- [x] Typography scale defined (12px-48px)
- [x] Touch target utility classes (.touch-target min 44px)
- [x] Animation utilities with GPU acceleration (transforms only, ≤200ms)
- [x] Cross-browser tested (Chrome, Safari, Firefox, Edge)

## Technical Details

### Files to Update
- `app/globals.css` - Primary design system definition

### Implementation Scope

#### 1. Spacing Tokens (8px Grid System)
Add CSS custom properties for consistent 8px-based spacing:
```css
--spacing-1: 8px;
--spacing-2: 16px;
--spacing-3: 24px;
--spacing-4: 32px;
--spacing-5: 40px;
--spacing-6: 48px;
--spacing-7: 56px;
--spacing-8: 64px;
```

#### 2. Typography Scale (12px-48px)
Define semantic typography tokens:
```css
--text-xs: 12px;
--text-sm: 14px;
--text-base: 16px;
--text-lg: 18px;
--text-xl: 24px;
--text-2xl: 32px;
--text-3xl: 48px;
```

Add corresponding line-height and letter-spacing for each scale.

#### 3. Utility Classes
Create practical CSS classes for common patterns:
- `.touch-target` - min-width/height 44px for interactive elements
- `.card-minimal` - padding, border-radius, subtle shadow
- `.btn-minimal` - padding, font-weight, transition
- `.text-truncate` - ellipsis for single-line text
- `.sr-only` - screen-reader-only content

#### 4. Animation Utilities
GPU-accelerated transitions (transforms and opacity only):
```css
.fade-in {
  animation: fadeIn 150ms ease-out;
  will-change: opacity;
}

.slide-up {
  animation: slideUp 150ms ease-out;
  will-change: transform;
}

@keyframes fadeIn {
  from { opacity: 0; transform: translateY(8px); }
  to { opacity: 1; transform: translateY(0); }
}

@keyframes slideUp {
  from { transform: translateY(16px); }
  to { transform: translateY(0); }
}
```

#### 5. Responsive Breakpoints
Leverage Tailwind defaults but document in globals.css:
```css
/* Mobile: 0-640px (sm: 640px) */
/* Tablet: 641-1024px (md: 768px) */
/* Desktop: 1025px+ (lg: 1024px) */
```

### Backward Compatibility
- Ensure all new tokens do not conflict with existing styles
- Preserve existing global styles
- Add new tokens as additional properties
