# Task: Migrate Sidebar/Navigation to CSS Modules

## Description
Migrate the sidebar navigation component (currently embedded in `app/page.tsx`) to use CSS Modules instead of Tailwind classes. Enhance with smooth transitions, improved hover states, and mobile-responsive behavior.

## Acceptance Criteria
- [ ] Create `components/Sidebar.module.css` with all sidebar styles
- [ ] Replace all Tailwind classes in sidebar JSX with CSS Module classes
- [ ] Implement smooth hover transitions (200ms ease-out)
- [ ] Add active navigation indicator with accent color
- [ ] Ensure 44x44px minimum touch targets for all nav items
- [ ] Mobile responsive: sidebar works at all breakpoints
- [ ] Playlist items have enhanced hover states
- [ ] No visual regression from current design

## Technical Details
**Files to create:**
- `components/Sidebar.module.css`

**Files to modify:**
- `app/page.tsx` (sidebar section, lines 110-141)

**Implementation approach:**
1. Create Sidebar.module.css with these class names:
   - `.sidebar` - main container
   - `.logo` - Doit Music heading
   - `.nav` - navigation container
   - `.navLink` - navigation links
   - `.navLinkActive` - active state
   - `.playlistSection` - playlist container
   - `.playlistItem` - playlist items
   - `.playlistItem:hover` - hover state

2. Replace Tailwind classes in page.tsx:
   - `className="w-64 flex flex-col"` → `className={styles.sidebar}`
   - `className="flex items-center space-x-3"` → `className={styles.navLink}`
   - etc.

3. Add CSS enhancements:
   - Hover transitions: `transition: all 200ms ease-out;`
   - Touch targets: `min-height: 44px; min-width: 44px;`
   - Active indicator: `border-left: 3px solid var(--primary);`

4. Implement responsive behavior:
   ```css
   @media (max-width: 640px) {
     .sidebar { /* Mobile styles */ }
   }
   ```

**Code locations affected:**
- `app/page.tsx` lines 110-141 (sidebar section)
